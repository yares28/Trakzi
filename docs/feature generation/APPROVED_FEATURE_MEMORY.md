# Approved Feature Memory

This file contains feature concepts that are already approved for implementation discussion or prototyping.

Do not re-propose these unchanged.

---

## Approved Features

| Feature Name | Type | Difficulty | Primary Surface | Existing Surface(s) Used | User Problem Solved | What It Does | Why It Matters | MVP Shape | New Data / Infra Needed | Confidence |
|---|---|---|---|---|---|---|---|---|---|---|
| Grocery Budget Guardrail | Cross-Feature | Medium | Fridge | Fridge x Analytics budgets x Savings | Users can analyze grocery behavior after the fact, but Trakzi does not warn when current shopping patterns are about to break the grocery budget. | Uses current visit cadence, basket size, and remaining month budget to flag whether the user is pacing toward an overrun. | It connects fridge intelligence to budget control and follow-through. | A guardrail strip on Fridge with projected month-end grocery spend and quick cutback suggestions. | Medium - pacing projections that join fridge receipt data with budget settings. | High |
| Chat Answer Evidence Drawer | Strengthen Existing | Medium | AI Chat | Chat x Analytics bundle x Fridge bundle x Savings bundle | Users can get chart-backed answers, but they still cannot quickly inspect what data window or source charts produced the answer. | Adds an expandable evidence drawer on AI replies showing source surfaces, date range, and the key stats or charts used. | It improves trust, debuggability, and follow-through for AI answers. | Expandable evidence section below eligible assistant messages with source pills and deep links. | Light - answer metadata from existing bundle inputs and link targets. | High |
| Limit Forecast Banner | Cross-Feature | Medium | System | Limits x Transactions wallet x AI chat usage x Categories x Billing | Users often discover plan or usage limits only when writes are blocked or a paid action fails. | Forecasts whether the user is likely to hit transaction, chat, or taxonomy limits soon and points to the cheapest next action. | It improves monetization and reduces frustrating hard-stop moments. | Banner and detail sheet showing projected limit exhaustion and suggested remediation. | Medium - forecast heuristics across existing usage and subscription data. | High |
| SSO Resume Destination Guard | Strengthen Existing | Easy | System | Sign-in x Sign-up x SSO callback x Home x Analytics x Fridge x Rooms | Users who authenticate through SSO can lose the page or task they were trying to complete, especially on deeper product routes. | Stores the blocked destination plus one pending action token and resumes the exact page/task after the auth callback completes. | It removes a fragile seam in activation and recovery flows. | Route resume plus a lightweight pending-action token for upload, review, or room actions. | Light - callback routing state and pending-action storage. | High |
| Billing Return Recovery Hub | Strengthen Existing | Easy | System | Billing return x Subscription status x Pricing | After checkout or billing changes, users can land on `/billing/return` without clear guidance on what succeeded, failed, or what to do next. | Turns the return page into a recovery hub that explains billing outcome, plan state, and the next valuable action in-app. | It makes monetization flows feel more reliable and reduces confused post-payment drop-off. | Outcome card on `/billing/return` with status, plan delta, retry actions, and open-the-right-page CTA. | Light - richer billing return state handling from existing subscription endpoints. | High |
| Room Provenance Reconciliation | Cross-Feature | High | Rooms | Rooms x Receipt OCR x Statement import x Data Library | Shared rooms mix manual entries, receipts, and imported bank transactions, so users can lose trust in whether a spend is counted once, twice, or not at all. | Adds provenance badges and duplicate-risk detection so room transactions show where they came from and what should be merged or replaced. | It makes room balances easier to trust and lowers reconciliation friction in shared-expense flows. | Provenance badges on room transactions with duplicate warnings and merge / replace suggestions. | Medium - source-link metadata and duplicate detection across room-linked records. | High |
