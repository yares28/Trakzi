# Trakzi / Folio — High-Fidelity Design System

Extracted from: **Landing Page** (`app/(landing)/page.tsx`) and **Analytics Page** (`app/analytics/_page/AnalyticsPage.tsx`)
CSS sources: `app/globals.css`, `app/(landing)/landing.css`

---

## Design Context

### Users
Financial power-users and everyday budget-trackers who import bank CSVs, scan receipts, and want rich visual breakdowns of their spending. They care about clarity, precision, and data density without sacrificing aesthetic quality.

### Brand Personality
**Precise. Warm. Confident.**
Trakzi uses warm neutrals (creamy off-white, warm darks) punctuated by a single expressive orange accent. It feels serious enough for financial data but human enough to be enjoyable.

### Aesthetic Direction
- Landing page: full dark-mode, cinematic, high-motion (Framer Motion)
- App (analytics): light + dark, minimal chrome, data-first, glass UI accents on mobile
- Reference feel: Linear + Vercel (precision) + Figma (fluidity) — but warmer
- Anti-reference: cold/clinical enterprise dashboards, neon fintech

### Design Principles
1. **Orange as the single accent** — brand color `#e78a53` appears on CTAs, hover states, focus rings, scrollbar active, cursor, text selection, and chart highlights. Never dilute it.
2. **Motion is intentional** — entrance animations stagger with 0.1s delays; hover transforms are 200-300ms; nothing surprises or distracts.
3. **Container-aware responsiveness** — use `@container` queries for components that render in different layout contexts (sidebar, grid, etc).
4. **Dark mode is semantic** — landing page forces dark; app respects user preference. The token system maps to completely different raw values per mode.
5. **Data density with breathing room** — stats cards are compact but padded; charts have enough whitespace to read clearly.

## Page-Specific Notes

### Landing Page
- **Always dark mode** — forced via `setTheme("dark")` on mount, restored on unmount
- Landing-page-active flag stored in `localStorage` key `trakzi-landing-active`
- User's theme saved to `trakzi-user-theme-before-landing` and restored on navigate away
- Navigation is a floating pill that narrows on scroll (`window.scrollY > 100`)
- All scroll-to-section logic uses `headerOffset = 120` to account for sticky header height
- Background logo: `/Trakzi/fulleticonB.svg` — three layered images with different blend modes
- Social proof partner logo: `/logoYdark.png`

### Analytics Page
- **Light + dark mode** — user preference respected
- `viewMode` state: `"analytics" | "advanced" | "trends"` — synced with `?view=` URL param
- `showEffectiveCosts` persisted to `localStorage` key `analytics:showEffectiveCosts`
- File drop: entire `<main>` is the drop zone (drag events bubble up)
- Drag indicator: `html[data-file-dragging="true"]` — scrollbars hidden globally
- Onboarding tour: `<OnboardingTour pageId="analytics" />` mounts on the page
- All data via single bundle API (`/api/charts/analytics-bundle`) + Redis cache
- React Query key: `["analytics-bundle"]`
- Advanced charts gated behind `planFeatures.advancedChartsEnabled`

## Accessibility

- `@media (prefers-reduced-motion: reduce)` — disables `chart-content-enter` animation and polar bar transitions
- Focus ring: `outline-ring/50` globally; ring color = `var(--primary)` (orange)
- `tabindex` on interactive elements inherits hand cursor
- Touch targets: min 44×44px on mobile
- Cards use `aria-*` where appropriate (drag handles, upload zones)
- Scrollbar gutter stable to prevent layout shift

---

## What This Design System Excludes

- **Color palette provider** (`components/color-scheme-provider.tsx`) — 12 named chart palettes, use `useColorScheme().getPalette()` — never hard-code chart colors
- **Currency formatting** — use `useCurrency().formatCurrency()` — never hard-code currency symbols
- **Chart component patterns** — each chart: `React.memo` + `displayName` + `LazyChart` wrapper for below-fold rendering
- **Sortable grid (dnd-kit)** — positions saved to localStorage, synced on mount
